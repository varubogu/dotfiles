# kanata 設定の思想

## 概要

このkanata設定は、**ホームポジションを崩さずにEmacsキーバインドを使う**ことを目的としています。

キーボードから手を大きく動かさなくてもカーソル移動・編集操作ができるよう、CapsLockをモディファイヤとして活用します。

---

## 設計の基本方針

### 1. 単押し / 長押し の二重割り当て

`tap-hold` を活用し、1つのキーに2つの役割を持たせています。

| キー | 単押し | 長押し |
|------|--------|--------|
| CapsLock | Esc | Emacs-Cレイヤー（C-x系、有効中） |
| 無変換 | 無変換（IME切替） | Emacs-Mレイヤー（M-x系、有効中） |
| 変換 | 変換（IME切替） | Winキー（Win+α も可） |

- **CapsLock単押し → Esc**：Vimユーザーにも自然に使えるよう、よく使うEscを近い位置に配置。
- **CapsLock長押し → Emacs-Cレイヤー**：押している間だけレイヤーが有効になり、ホームポジション付近のキーがC-x系のカーソル移動・編集操作に変わる。
- **無変換長押し → Emacs-Mレイヤー**：押している間だけレイヤーが有効になり、各キーがM-x系（Alt+キー）のEmacs操作に変わる。
- **変換長押し → Winキー保持**：長押しのまま離すとWinキー単体押下扱い（スタートメニュー等）、長押ししながら別キーでWin+αのショートカットになる。

### 2. Emacs-Cレイヤーのキーマップ（CapsLock長押し）

CapsLockを押しながら以下のキーを使います。
※CapsLockキーは完全に潰しています。

| キー | 機能 | Emacsコマンド相当 |
|------|------|-------------------|
| A | Home（行頭） | C-a |
| B | ← 左移動 | C-b |
| F | → 右移動 | C-f |
| P | ↑ 上移動 | C-p |
| N | ↓ 下移動 | C-n |
| E | End（行末） | C-e |
| D | Delete（後ろ削除） | C-d |
| H | Backspace（前削除） | C-h |
| K | 行末まで削除（S-End + Backspace） | C-k |
| Y | ペースト（Ctrl+V） | C-y（yank） |
| V | PageDown | C-v |
| X | C-x プレフィックスモードへ | C-x |

### 3. Emacs-Mレイヤーのキーマップ（無変換長押し）

無変換キーを押しながら以下のキーを使います。各キーはAlt+キー（Emacsでのメタキー相当）を送出します。

| キー | 機能 | Emacsコマンド相当 |
|------|------|-------------------|
| A | backward-sentence | M-a |
| B | backward-word（単語左移動） | M-b |
| F | forward-word（単語右移動） | M-f |
| P | 前候補（ミニバッファ等） | M-p |
| N | 次候補（ミニバッファ等） | M-n |
| E | forward-sentence | M-e |
| D | kill-word（単語削除） | M-d |
| H | mark-paragraph | M-h |
| K | kill-sentence | M-k |
| Y | yank-pop | M-y |
| V | scroll-down（PageUp相当） | M-v |
| X | execute-extended-command | M-x |
| BackSpace | backward-kill-word（単語後退削除） | M-BackSpace |

### 4. C-x プレフィックスレイヤー

`CapsLock + X` でC-xプレフィックスレイヤーに入ります（`one-shot-press` で5秒間有効）。

| キー | 機能 | Emacsコマンド相当 |
|------|------|-------------------|
| S | 保存（Ctrl+S） | C-x C-s |

### 5. 全機能バイパス

`CapsLock長押し + BackSpace` で**標準モード（bypass-mode）**に切り替えられます。

バイパスモードでは、kanataによるすべてのキーリマップが無効になりOSの標準動作がそのまま通ります。ゲームやリマップと競合するアプリで利用します。もう一度同じ操作で通常のEmacsモードに戻ります。

---

## キー選定の理由

- **CapsLockをモディファイヤに選んだ理由**：普段ほぼ使わないキーであり、ホームポジションに近いため押しやすい。
- **無変換をEmacs-Mレイヤーに選んだ理由**：日本語環境でAltキーは離れた位置にあるため、スペースバー左隣の無変換でM-x系操作を行えるようにする。左親指で押せる位置に配置。
- **変換をWinキーに選んだ理由**：スペースバー右隣の変換キーをWinキー代替とすることで、右親指でWindowsショートカット（Win+L、Win+Dなど）を扱いやすくする。
- **`defsrc` に最小限のキーだけ登録している理由**：kanataが介入するキーを最小限にとどめ、他のキーへの影響を防ぐため。

# Bilingual Documentation Sync

## When to use

Use when creating or updating documentation that has two language versions.

## Workflow

1. Identify the source document and its paired language counterpart.
2. Apply equivalent structural/content updates to both files.
3. Verify headings, examples, and command snippets are aligned.
4. If intentional differences exist, document the reason explicitly.

## Quality checks

- Keep terminology consistent across both versions.
- Ensure links point to corresponding language paths when available.
- Avoid updating only one side unless the user explicitly requests it.

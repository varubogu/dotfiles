# Rust テスト

## 使用タイミング

Rust のテスト作成、失敗原因の切り分け、回帰防止の作業で使用する。

## 実行戦略

1. まず対象を絞ったテスト（特定 crate/ファイル/テスト名）を実行する。
2. 範囲を広げる前に根本原因を修正する。
3. 対象テストが通った後に、関連する全体スイートを実行する。

## テスト設計

- ユニットテスト: ビジネスロジックとエラーブランチを分離して検証する。
- 統合テスト: 現実的なセットアップでレイヤー横断の挙動を検証する。
- アサーションは明示的にする（成功系 + 失敗系）。

## 信頼性メモ

- 決定的なテストデータを使い、共有可変状態を避ける。
- ローカルマシン負荷が懸念される場合は worker 数を制御する（例: `cargo test -j 4`）。

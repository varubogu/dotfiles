# フロントエンドテスト（Vitest）

## 使用タイミング

フロントエンドのユニットテスト、コンポーネントテスト、統合テストの実装やデバッグ時に使用する。

## 推奨フロー

1. 影響範囲の単一テストファイルを先に実行する。
2. アサーション、モック、型の問題を最小変更で修正する。
3. 同じファイルを安定するまで再実行する。
4. 局所的な安定を確認してから、より広いテスト範囲を実行する。

## パターン

- Arrange-Act-Assert 構造を優先する。
- テストを独立させる（`beforeEach` でリセット/セットアップ）。
- 外部依存は安定した境界でモックする。
- コンポーネントではまずユーザー観測可能な挙動をテストする。

## よくある確認事項

- 終了前に型チェックと lint を実行する。
- 不安定な失敗が出る場合、時間依存の前提を除き、最終状態を検証する。

# タスク実行ワークフロー

## 使用タイミング

実装、リファクタリング、バグ修正、テスト更新を依頼されたときに使用する。

## ワークフロー

1. 要求される挙動と制約を明確化する。
2. まず関連ファイルのみを確認し、広範囲な編集を避ける。
3. 要求を満たすための最小変更を実装する。
4. 広いスイート実行前に、対象を絞ったチェック/テストで検証する。
5. 何を、なぜ変更し、どう検証したかを報告する。

## ガードレール

- ユーザー承認なしに無関係なコードを編集しない。
- 新規パターン導入より既存アーキテクチャ/スタイルを優先する。
- 仮定が必要な場合は明示する。
- 検証を完了できない場合は、残作業を具体的に示す。
